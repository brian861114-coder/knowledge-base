# Schema-First Workflow

This template assumes that structure is a machine-checkable contract.

## The Core Rule

Models do not decide the page skeleton.

The schema decides:

- note types
- section names
- section order
- conditional rules
- what counts as optional

## Required Loop

1. Schema
2. Skeleton
3. Content fill
4. Structure validation
5. Content audit
6. Export
7. Full validation

## Why This Exists

Without a schema-first loop, AI output drifts into:

- inconsistent headings
- arbitrary order
- missing sections
- template filler
- fragile migrations

The schema-first loop is the fix.
