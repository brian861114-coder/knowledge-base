目前這個原型已完成模組拆分與資料夾整理。入口仍是根目錄的 app.js， 但功能模組已集中到 src/，測試移到 tests/，腳本移到 scripts/， 文件移到 docs/。

2. 檔案分工

入口層

index.html 提供 SVG 舞台、detail panel、toolbar 與 minimap 容器。

app.js 負責初始化、狀態協調、事件綁定與整體 render orchestration。

資料與驗證

src/logic.mjs 負責 graph/detail payload 驗證、關係方向分類、搜尋匹配、detail 檔名計算。

src/detail-store.mjs 負責 detail index 與 detail payload 的 lazy loading、快取與錯誤狀態。

場景與幾何

src/scene-builder.mjs 生成 overview/focus scene data。

src/scene-geometry.mjs 與 src/layout-solver.mjs 處理邊路徑、環狀排布、碰撞放鬆。

介面與渲染

src/detail-renderer.mjs 負責 detail view 與全文/摘要區塊 HTML。

src/detail-panel.mjs 負責 pills、meta、stat cards、search result HTML。

src/ui-state.mjs 負責 breadcrumb 與 mode UI state。

互動支援

src/navigation-state.mjs 管理 overview/focus 狀態切換與瀏覽歷史。

src/viewport.mjs 管理 pan/zoom/minimap 轉換與邊界限制。

文本處理

src/markdown.mjs 處理 markdown/wikilink 渲染。

src/mathjax.mjs 管理數學公式 typeset 排程。

3. 主要資料流

初始化流程：

index.html
  -> app.js init()
  -> fetch ../physics_graph.json
  -> validateGraphPayload()
  -> normalizeGraph()
  -> createDetailStore()
  -> loadIndex(./data/detail-index.json)
  -> buildOverviewScene()
  -> bindEvents()
  -> render()

點擊節點後的 detail 流程：

select node
  -> createFocusState()
  -> buildFocusSceneData()
  -> renderScene()
  -> renderDetail()
  -> if detail not cached: loadDetail()
  -> buildNoteSectionHtml()

4. 目前架構上的實際狀態

項目 狀態 說明
模組拆分 已完成主要拆分 資料驗證、scene builder、navigation、UI renderer、viewport helper 都已分離。
資料夾整理 已完成 根目錄不再塞滿所有模組與測試檔。
單元測試 可用 目前使用 Node 內建測試，模組級測試通過。
入口檔大小 仍偏大 app.js 仍負責太多 orchestration，尚未完全瘦身。
瀏覽器級 QA 不足 目前以 node --check 和 node --test 為主，尚缺完整 browser smoke test。

5. 測試與驗證方式

目前可靠的本地驗證指令：

node --check prototype_semantic_lens/app.js
node --check prototype_semantic_lens/scripts/generate_detail_payloads.mjs
node --test prototype_semantic_lens/tests/*.test.mjs

在目前這個靜態前端原型中，這些指令是最低成本且最穩定的驗證關卡。

6. detail payload 產生方式

scripts/generate_detail_payloads.mjs 目前會：

  1. 讀取 repo 根目錄的 physics_note_details.json
  2. 驗證 payload key 與結構
  3. 輸出到根目錄的 data/detail-index.json 與 data/details/*.json
node prototype_semantic_lens/scripts/generate_detail_payloads.mjs

7. 已知限制

  • app.js 仍然偏胖，尚未完全收斂成純 orchestration。
  • 搜尋區塊附近仍有一段未清除的 dead code，現在不影響執行，但仍屬技術債。
  • 這個原型先前曾出現 viewport regression、detail 404、SVG NaN 類問題；模組測試目前是綠的，但仍應用瀏覽器層 QA 驗證實際互動。
  • 根目錄文字顯示有部分編碼觀察差異，文件與程式碼編輯時不應輕率做大面積文字替換。

8. 建議的後續維護原則

後續新增功能時，建議遵守這幾條：

入口薄化資料邏輯放 src/logic.mjsUI HTML 放 renderer先加測試再擴功能

  • 搜尋、detail、breadcrumb 這類 UI 字串組裝不要再回流到 app.js。
  • scene 計算與 viewport 數學維持純函式，可測試、可重用。
  • 每次抽模組或搬檔案後，都至少重新跑一次 node --check 與 node --test。
